Logistic regression is a **classification** algorithm despite the name. It runs a linear combination of features through a sigmoid to squash the output into `[0,1]`, giving `P(y=1|x) = 1 / (1 + e^{-(β₀ + βᵀx)})`, then thresholds that probability into a class.

The differences from linear regression are real, not cosmetic. Linear regression outputs an unbounded continuous number and minimizes squared error; logistic outputs a probability and minimizes **log-loss (cross-entropy)**. You can't use least squares here — squared error on a sigmoid is non-convex and behaves badly, whereas log-loss is convex and trains cleanly with gradient descent. The decision boundary is still linear in feature space, so logistic regression can't separate XOR-like patterns without engineered features.

The coefficients are interpretable as **log-odds**: `exp(βᵢ)` is the odds ratio for a one-unit change in `xᵢ`, which stakeholders in regulated settings (credit, healthcare) love because you can defend every decision. It extends to multiclass via softmax.

Two production notes. First, the default 0.5 threshold is almost never what you want — tune it to the cost of false positives versus false negatives. Second, on imbalanced data (fraud, rare disease) the raw model is often poorly calibrated and biased toward the majority class; use class weights and check a reliability curve before trusting the probabilities.
