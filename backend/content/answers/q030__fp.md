The problem: we want to predict a *probability* of class membership, a number confined to [0, 1]. Linear regression outputs any real number, so it will happily predict 1.7 or −0.3 — nonsense as probabilities, and its Gaussian-noise assumption is wrong for binary outcomes. We need to respect the geometry of the output.

The elegant fix: keep the linear model, but predict on the right scale. Model the **log-odds** — log(p/(1−p)) — as linear in the features. Log-odds range over all reals, so a linear predictor fits naturally. Inverting gives the **sigmoid**, which squashes any real input into (0, 1). So logistic regression is a linear model *in log-odds space*, bent through a nonlinearity to land in probability space.

Why the sigmoid specifically? It's the canonical link for the Bernoulli distribution in the exponential family — the mathematically "natural" choice, and it makes the log-likelihood concave, so optimization has a single global optimum.

Key differences:

- **Output**: continuous value vs. probability → class.
- **Loss**: squared error (Gaussian MLE) vs. cross-entropy / log-loss (Bernoulli MLE).
- **Fitting**: closed-form normal equations vs. iterative gradient methods (no closed form).

The non-obvious insight: both are the *same idea* — maximum likelihood under a linear predictor — differing only in the assumed noise distribution. Change Gaussian to Bernoulli and least-squares becomes logistic regression. Both are generalized linear models.
