question: "What is the difference between a primary key and a foreign key?"
topic: "Data Analytics"
model: "Claude Code (authored)"
sources: []
---

Think of a school. Every student gets a unique ID number, no two students share it and nobody is left without one. That's a **primary key**: the column that uniquely names each row in a table, never repeated, never blank.

Now imagine the library keeps a list of borrowed books, and each loan writes down the borrower's student ID. That borrowed-ID is a **foreign key**: it points back to a real student in the students table. It keeps things honest, you can't loan a book to a student ID that doesn't exist.

The differences: a primary key must be unique and filled in; a foreign key can repeat (one student borrows many books) and can even be blank (a book nobody borrowed). Together they link tables so you can **join** a loan back to the person who made it.
