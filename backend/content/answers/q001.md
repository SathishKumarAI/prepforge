Think of them as three nested circles. **AI** is the outer circle: any system that does something we'd call "intelligent" — a rules engine that plays chess, an A* pathfinder, a fraud-detection expert system. No learning required. **Machine learning** is the middle circle: systems that improve at a task by fitting parameters to data instead of being hand-coded. **Deep learning** is the inner circle: ML using multi-layer neural networks that learn their own feature representations.

The practical distinction I lean on in interviews is *where the intelligence comes from*. In classic AI, an engineer encodes it as rules. In ML, it's learned from labeled examples but you still hand-craft features (TF-IDF, aggregations). In DL, the network learns the features too, which is why it dominates unstructured data — images, audio, text — where hand-crafting features is hopeless.

A concrete production lens: for tabular data I still reach for gradient-boosted trees (ML, not DL) because they're faster to train, easier to explain, and usually win. I only pay the DL tax — GPUs, data hunger, longer iteration loops — when the input is unstructured. So "every DL model is ML, every ML model is AI, not vice versa" is correct, but the reverse question — *do you even need DL?* — is the one that matters on the job.
