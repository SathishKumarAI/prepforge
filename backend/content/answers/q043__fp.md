Ask a sharper question: when does the *unit* of a feature change the answer? If an algorithm's decisions depend on how you measure a variable (dollars vs. cents, meters vs. millimeters), it needs scaling. Scaling — standardization to zero mean/unit variance, or min–max to `[0,1]` — removes that arbitrary unit dependence.

The principle is that many algorithms implicitly assume a **geometry**. Anything using distances (`k`-NN, `k`-means, RBF-kernel SVM) computes `‖x − y‖`, which is dominated by whichever feature has the largest numeric range — income (thousands) drowns age (tens). Scaling makes the metric isotropic so each dimension gets a fair vote.

Gradient-based methods need it too, but for a different reason: unequal feature scales make the loss surface an *elongated ellipse*. Gradient descent then zig-zags across the narrow valley, and a single learning rate can't fit both axes. Scaling makes the contours round, so descent goes straight to the minimum — dramatically faster convergence. Regularization also assumes comparable scales, since one penalty `λ` is shared across coefficients.

Non-obvious insight: **tree-based models don't care.** Splits depend only on *rank order* within a feature, which any monotonic rescaling preserves. Scaling is about geometry, and trees have none.
