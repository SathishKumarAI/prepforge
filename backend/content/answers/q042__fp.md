question: "How does gradient boosting differ from bagging (random forests)?"

Both are ensembles, but they attack opposite terms of the error decomposition. Bagging averages many *independent, low-bias, high-variance* trees to cancel variance. Boosting builds many *dependent, high-bias, low-variance* stumps sequentially to cancel bias.

Derive boosting from optimization. You want to minimize a loss `L(y, F(x))` over the *function* `F`. Do gradient descent in function space: at each step compute the negative gradient of the loss with respect to the current prediction — for squared error this is just the residual `y − F(x)` — and fit a small tree to *approximate that gradient*. Then step: `F ← F + η·h`. Each tree is a descent direction; the learning rate `η` is the step size. That is literally gradient descent, but the "parameter" is a function and each update is a regression tree.

This explains every difference. Bagging is embarrassingly parallel (order-independent); boosting is inherently sequential (each tree corrects the last). Bagging can't reduce bias; boosting can drive training error to zero, so it *can overfit* and needs regularization (shrinkage, depth limits, subsampling).

Non-obvious insight: random forests average, boosting *adds*. One is a mean, the other is a Taylor-style expansion of the target.
