The goal of learning is *generalization* — low error on data we haven't seen. But we can only ever measure error on data we *have* seen. Overfitting is the gap that opens when a model drives training error down by memorizing the **noise and idiosyncrasies** of the sample rather than the **signal** shared with the population.

Why must this happen? Any finite dataset = true pattern + sampling noise. A sufficiently flexible model can fit both perfectly. But the noise won't repeat in new data, so fitting it *increases* test error. This is the bias–variance decomposition made concrete: as capacity grows, bias falls but variance (sensitivity to the particular sample) rises, and total error is U-shaped.

Prevention all attacks the same lever — **reduce effective capacity relative to the information in the data**:

- More/augmented data (dilutes noise, strengthens signal).
- Regularization (L1/L2, dropout) — penalize complexity, shrinking the hypothesis space.
- Early stopping — halt before the model starts fitting noise.
- Simpler models or fewer features.
- Cross-validation — *detect* the gap so you can tune.

The unifying principle: generalization is possible only when the hypothesis space is small relative to the data (Occam's razor / statistical learning theory).

Non-obvious insight: overfitting isn't caused by big models per se, but by *capacity exceeding the data's information content* — huge models generalize fine when data and regularization scale with them.
