Machine learning models only speak numbers, so word-categories like "red, green, blue" must be translated first. The trick is translating *without accidentally lying* to the model.

**One-hot encoding** gives each category its own yes/no column — one box ticked for "red," another for "blue." It's perfect for unordered labels because it never implies red is somehow bigger than blue. The cost: lots of columns when there are many categories.

**Label encoding** just numbers them 1, 2, 3. That's fine when there's a real order (small, medium, large), but dangerous for things like colors, since the model would wrongly think blue (3) outranks red (1).

For categories with tons of values, **target encoding** replaces each with the average outcome for that group — powerful, but it can leak answers and overfit, so use it carefully. Pick your translation based on whether the categories have a natural order and how many there are.
