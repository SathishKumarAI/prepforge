question: "How does a random forest work and why is it better than a single decision tree?"

Start from the failure mode of a single tree: it is a high-variance estimator. Because it splits greedily and recursively, tiny changes in the data reshuffle every downstream split, so the tree memorizes noise. Its bias is low but its variance is enormous.

The deeper principle is that averaging *reduces variance*: for `n` estimators each with variance `σ²` and pairwise correlation `ρ`, the average has variance `ρσ² + (1−ρ)σ²/n`. The `(1−ρ)/n` term vanishes with more trees, but the `ρσ²` floor does not. So the whole game is **decorrelation**.

A random forest attacks both terms. Bagging (bootstrap samples) gives each tree different data. The clever extra move is sampling a random *subset of features* at every split, which forces trees to disagree — lowering `ρ` — even when one feature dominates. That is why it beats plain bagging.

The non-obvious insight: you deliberately grow each tree *worse* (weaker, more random) to make the ensemble *better*, because you are buying independence. Individual accuracy is sacrificed for statistical independence, and the average redeems it.
