question: "What is the difference between classification and regression?"

Both are supervised learning — learn a function *f(x) → y* from examples — so the real question is: **what is the geometry of the output space y?**

In **regression**, y is continuous and ordered: a number line where distance and direction are meaningful (predicting 3.1 when the truth is 3.0 is *nearly right*). This ordering dictates everything downstream. The natural loss is squared or absolute error, because "how far off" is well-defined. Under a probabilistic lens, regression models a continuous conditional distribution (often Gaussian), and minimizing squared error is maximum likelihood for that assumption.

In **classification**, y is a discrete set of categories with *no inherent metric*: predicting "cat" when the truth is "dog" is simply wrong — there's no notion of "close." So squared error is nonsensical; instead we predict a probability distribution over classes and minimize cross-entropy, which measures the information cost of the wrong distribution.

The unifying principle: your **loss function must respect the structure of the output space**. Order → distance-based loss; unordered categories → likelihood/information-based loss.

The non-obvious insight: the boundary blurs. Logistic "regression" is classification; ordinal targets sit between the two; and you can always discretize a regressor or treat class indices as numbers — but doing so *imposes a metric that may not exist*, which is exactly the mistake to avoid.
