Begin with the fundamental problem: light bounces off a 3D world and lands on a 2D sensor as an array of numbers. Computer vision is the **inverse problem** of recovering meaningful structure — objects, geometry, motion, semantics — from that array. It's hard precisely because the projection is lossy and ambiguous: infinitely many scenes can produce the same pixels, so vision is fundamentally about using **priors** to resolve ambiguity the way brains do.

The core tasks form a ladder of increasing abstraction:

- **Classification** — what is in the image? (one label for the whole grid)
- **Object detection** — what and *where*? (labels plus bounding boxes)
- **Semantic segmentation** — which class does *each pixel* belong to?
- **Instance segmentation** — separate individual objects of the same class.
- **Depth / pose estimation** — recover 3D geometry the projection discarded.
- **Tracking** — maintain identity across time.

The unifying principle: each task asks for a different *granularity of correspondence* between pixels and concepts, from one-label-per-image to one-label-per-pixel-per-instance-per-frame.

The non-obvious insight: these tasks aren't separate problems but points on a spectrum of spatial and temporal resolution, which is why modern architectures share backbones and differ only in their output heads.
