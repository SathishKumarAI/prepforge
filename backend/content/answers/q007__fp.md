A* solves a tension between two flawed strategies. Uniform-cost search (Dijkstra) orders nodes by g(n), the *known* cost so far — optimal but blind, expanding uselessly in all directions. Greedy best-first orders by h(n), the *estimated* cost remaining — fast but easily fooled into suboptimal paths. Each uses only half the available information.

A*'s insight is to add them: **f(n) = g(n) + h(n)** — the cost already paid plus the estimated cost to finish equals an estimate of the *total* path cost through n. Always expand the node with smallest f, using a priority queue as the frontier. You're betting on whichever partial path looks cheapest end-to-end.

Why is it optimal? If h is *admissible* (never overestimates), f is a lower bound on any complete path through n. So when A* pops a goal, no unexpanded node could possibly yield something cheaper — anything with smaller true cost would have had smaller f and been popped first. With a *consistent* h (triangle inequality), it never needs to re-expand a node.

The non-obvious insight: A* is provably *optimally efficient* — no algorithm using the same heuristic can expand fewer nodes. The heuristic's quality is everything: h=0 degenerates to Dijkstra; h=h* walks straight to the goal.
