A tree wants to carve feature space into regions that are as **pure** as possible — each region dominated by one class (or one output value). The engineering problem is: at each node, of all possible (feature, threshold) splits, which one to pick? The answer must be *greedy and local*, because finding the globally optimal tree is NP-hard, so the tree asks only: which single split most improves purity *right now*?

That requires quantifying "purity," and the natural measure is **impurity/uncertainty**. Two common ones:

- **Entropy** (from information theory) — the average bits needed to encode the class of a point. A pure node has zero entropy. Choosing the split that most reduces entropy = maximizing **information gain**, literally selecting the question that reveals the most about the label.
- **Gini impurity** — the probability of mislabeling a random point if you guessed by the node's class distribution. Cheaper to compute, behaves similarly.

For each candidate split, compute the *weighted* impurity of the resulting children, subtract from the parent's, and pick the maximum reduction. Repeat recursively.

The unifying principle: **splitting = greedy uncertainty reduction**, the same logic as playing optimal "twenty questions."

The non-obvious insight: this greedy criterion is myopic — it can't see a split that only pays off *after* a later one (like XOR), so trees miss interactions a global search would catch. That short-sightedness, plus unchecked growth overfitting, is exactly why **ensembles** (random forests, boosting) exist.
