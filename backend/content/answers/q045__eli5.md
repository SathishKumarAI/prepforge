question: "What is the difference between parameters and hyperparameters, and how do you tune hyperparameters?"
topic: "Machine Learning"
model: "Claude Code (authored)"
sources: []
---

Think of baking bread. Before you start, *you* choose the oven temperature, the baking time, and how much yeast — settings you dial in up front. During baking, the dough itself rises and browns on its own. The **parameters** are like that finished dough: values the model *learns automatically* from data, such as the weights in a neural network. The **hyperparameters** are the knobs *you* set beforehand — learning rate, number of trees, tree depth, the k in k-NN.

Since you can't learn hyperparameters straight from the data, you search for good ones. **Grid search** tries every combination on a list; **random search** samples combinations and is often faster; **Bayesian optimization** learns from past attempts to pick promising settings next. Always judge each setting on a validation set or cross-validation — never the test set — so your final performance estimate stays honest.
