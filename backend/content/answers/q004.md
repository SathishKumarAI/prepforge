An intelligent agent is anything that perceives its environment through sensors and acts on it through actuators to achieve goals. The classic loop is `percept → agent function → action`, and a *rational* agent picks the action expected to maximize its performance measure given what it knows.

The standard taxonomy scales by sophistication: simple reflex agents (condition-action rules on the current percept), model-based agents (maintain internal state about the world), goal-based agents (search/plan toward an objective), utility-based agents (optimize a utility function when trade-offs exist), and learning agents (improve from experience). The design axes that actually matter in practice are whether the environment is observable, deterministic, static, and single- vs multi-agent — those decide how much internal state and lookahead you need.

Today "agent" usually means an LLM wired to tools, memory, and a planning loop (ReAct-style reason-act-observe cycles, or plan-and-execute). The framing is identical: the model is the agent function, tool calls are actuators, tool results and retrieved context are percepts, and the objective is the user's goal.

In production the hard parts aren't the concept, they're the failure modes: agents loop forever, hallucinate tool arguments, or take irreversible actions. So I gate real-world side effects behind confirmation, cap step counts, and log the full percept-action trace. An agent with no guardrails and no observability is a liability, not autonomy.
