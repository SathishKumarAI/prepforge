question: "What is the difference between parameters and hyperparameters, and how do you tune hyperparameters?"

**Parameters** are what the model *learns* from data during training — the weights of a neural net, the coefficients of a linear model, the split thresholds in a tree. **Hyperparameters** are what *you* set before training and the algorithm doesn't touch: learning rate, number of trees, tree depth, regularization strength `λ`, `k` in k-NN, the number of layers. Rule of thumb — if gradient descent updates it, it's a parameter; if you configure it, it's a hyperparameter.

Tuning hyperparameters means searching that configuration space for the combo that generalizes best, always scored on a **validation set or via cross-validation** — never on training data (you'd just pick the most overfit config) and never on the test set (you'd contaminate your final estimate).

The main strategies, roughly in order of what I reach for:

- **Grid search** — exhaustive over a small discrete grid. Simple, but cost explodes combinatorially.
- **Random search** — samples the space; famously more efficient than grid when only a few hyperparameters actually matter (Bergstra & Bengio).
- **Bayesian optimization** — Optuna, Hyperopt; builds a surrogate model to propose promising points, best when each training run is expensive.

Two production habits: use **early stopping** to tune epochs for free, and don't over-tune — chasing the third decimal of validation AUC usually buys you noise, not generalization. Fix a reproducible search budget and stop when the curve flattens.
