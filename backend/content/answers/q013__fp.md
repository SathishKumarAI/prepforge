Start from what a trained LLM actually is: a fixed function that maps an input context to a distribution over continuations. You cannot change its weights, but you *can* change the input. **Prompt engineering** is the practice of shaping that input so the model's most probable continuation is the behavior you want. It is programming through natural language — the prompt is the program, and the weights are the interpreter.

Why does it matter so much? Because a large model contains a vast superposition of latent "skills" and personas absorbed during training. The prompt doesn't teach new abilities; it *selects and conditions* among ones already present — it navigates to a region of the model's behavior space. A vague prompt lands in an average, generic region; a precise one (clear role, constraints, examples, output format) collapses the distribution toward the competent behavior. This is why few-shot examples work: they let the model *infer the task* from pattern, and why chain-of-thought helps — asking for reasoning steps allocates more intermediate computation before the answer.

The non-obvious insight: prompting exploits the model's conditioning on *its own generated tokens*. Techniques like "think step by step" work not because the model is persuaded, but because generated reasoning tokens become context that mechanically improves the final prediction. You're steering computation, not motivating a mind.
