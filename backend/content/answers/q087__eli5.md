question: "What is the difference between a star schema and a snowflake schema?"
topic: "Data Analytics"
model: "Claude Code (authored)"
sources: []
---

Both are ways to arrange a data warehouse around a central "facts" table (the events you measure, like each sale) surrounded by "dimension" tables that add context (which product, which customer, what date).

Picture a star: one fact table in the middle, and each dimension is a single flat card connected straight to it, all the product info on one card. That's a **star schema**. Everything's close, so questions need few joins and run fast, but you repeat some details across cards.

A **snowflake schema** takes each card and breaks it into smaller linked cards: product points to category, which points to department. Less repeated info and less storage, but answering a question now means hopping across more tables (more joins), which is slower and fiddlier.

Rule of thumb: star is the popular default because it's simple and fast; snowflake suits big, deeply layered dimensions where saving storage matters more than speed.
