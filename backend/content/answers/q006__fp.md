question: "What is a heuristic in AI search, and what makes one admissible?"

Uninformed search is blind — it treats every unexplored node as equally promising, so it wastes effort exploring away from the goal. A **heuristic** injects domain knowledge: a cheap estimate h(n) of the remaining cost from node n to the goal. It's a *guess* that lets search prioritize nodes that look closer, focusing effort like a compass.

The subtle question is: when can a guess be trusted enough to still guarantee optimal answers? That's what **admissibility** formalizes. A heuristic is admissible if it *never overestimates* the true remaining cost: h(n) ≤ h*(n) for all n. It may under-guess (even h=0 is admissible), but it must never claim the goal is farther than it is.

Why must it work this way? An algorithm like A* orders nodes by f(n)=g(n)+h(n), the estimated total path cost. If h never overestimates, then f never overestimates the true cost of any path through n — so A* cannot commit to a suboptimal goal while a cheaper option still sits underexplored with a smaller f. Optimism guarantees optimality.

The non-obvious insight: admissibility is really a *lower-bound* guarantee. The closer h hugs h* from below, the fewer nodes expanded — but overshoot by even a little and optimality can break. Accuracy and safety pull in opposite directions.
