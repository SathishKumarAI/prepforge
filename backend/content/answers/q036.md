Both add a penalty on the weights to the loss to fight overfitting, but the *shape* of the penalty changes the behavior. **L1 (Lasso)** adds `λ·Σ|wᵢ|`; **L2 (Ridge)** adds `λ·Σwᵢ²`.

The key practical difference: **L1 produces sparsity — it drives some weights to exactly zero**, effectively doing feature selection. L2 shrinks all weights smoothly toward zero but essentially never *to* zero. The geometric intuition that makes this click: L1's constraint region is a diamond with corners on the axes, and the optimum tends to land on a corner (a zero coordinate); L2's region is a smooth circle with no corners, so it just shrinks.

When I choose: **L1** when I suspect many features are irrelevant and I want a sparse, interpretable model — it's a built-in feature selector. **L2** when features are correlated and I'd rather share weight across them than arbitrarily zero some out; it's also smoother to optimize (differentiable everywhere). **Elastic Net** combines both and is my default for high-dimensional correlated data — it gets Lasso-style sparsity without Lasso's habit of randomly picking one feature from a correlated group and dropping the rest.

`λ` controls strength: larger λ → more shrinkage → higher bias, lower variance. Tune it by cross-validation. And a gotcha I've watched bite people: regularization penalizes weights by magnitude, so you **must scale/standardize features first** or the penalty falls unevenly on features with larger units.
