Start from what a model actually does: it minimizes error on a training distribution. Bias is not a bug layered on top — it's the *faithful* consequence of that objective when the data or the objective misrepresent the world you care about.

Trace it to sources along the pipeline:

- **Sampling/representation bias** — the training data isn't drawn from the deployment population (underrepresented groups get higher error).
- **Historical bias** — the data accurately reflects an unjust world; the model learns to reproduce it (past hiring favored one group → model favors it).
- **Label bias** — the targets encode subjective or prejudiced human judgments.
- **Measurement/proxy bias** — a convenient proxy (e.g. arrests as a proxy for crime) is itself skewed.
- **Aggregation bias** — one model forced onto subpopulations that need different treatment.

The unifying principle: a model can only be as fair as the **joint distribution it was optimized against**; if that distribution differs from your target of fairness, no amount of optimization fixes it.

Remedies attack the source: audit and rebalance data, define fairness metrics explicitly (they conflict — you must choose), reweight or add constraints, and monitor post-deployment.

Non-obvious insight: impossibility results prove several intuitive fairness criteria are **mathematically incompatible** at once, so fairness is a value choice, not a computation.
