The core problem in language: a word's meaning depends on *other* words, arbitrarily far away. "It" refers to something; "bank" is disambiguated by distant context. Earlier architectures (RNNs) passed information sequentially, so distant dependencies degraded and computation couldn't parallelize. Self-attention asks a sharper question: for each token, *which other tokens should I look at, and how much?*

The mechanism answers it with a learned **content-based lookup**. Each token emits three vectors: a **query** (what I'm looking for), a **key** (what I offer), and a **value** (what I'll contribute). The relevance of token j to token i is the dot product of i's query with j's key — high when they're semantically aligned. Softmax over these scores turns them into weights, and each token's new representation is the weighted sum of all values. Every position gathers exactly the context it needs, in one step.

Why this works: it's a **differentiable, soft dictionary** — a routing of information determined by the data itself, not by fixed positions. And because every pair interacts directly, path length between any two tokens is O(1), so gradients and information flow freely, and the whole operation is a matrix multiply — massively parallel.

The non-obvious insight: attention has *no inherent notion of order* (it's permutation-equivariant), which is why transformers must inject **positional encodings** separately. Sequence structure is bolted on, not built in.
