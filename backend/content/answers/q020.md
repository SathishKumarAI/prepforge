Q-learning is a model-free, off-policy RL algorithm that learns the optimal action-value function `Q(s,a)` — the expected discounted return of taking action `a` in state `s` and then acting optimally forever after. Once you have `Q*`, the optimal policy is trivial: in each state pick `argmax_a Q(s,a)`.

It learns by bootstrapping from experience with the temporal-difference update:

`Q(s,a) ← Q(s,a) + α [ r + γ·max_a' Q(s',a') − Q(s,a) ]`

The bracket is the TD error: the difference between the observed target `r + γ·max Q(s',a')` and the current estimate. `α` is the learning rate, `γ` the discount. The `max` over next-state actions is what makes it **off-policy** — it learns about the greedy policy while the agent actually behaves with something exploratory like ε-greedy, so you can learn the optimum from suboptimal, exploratory data. With sufficient exploration of every state-action pair and a decaying learning rate, tabular Q-learning provably converges to `Q*`.

Tables don't scale past small discrete spaces, so in practice you approximate `Q` with a neural net — **DQN**. Naively that diverges, which is why DQN adds two stabilizers I've seen matter enormously: an **experience replay** buffer (break correlation between consecutive samples) and a **target network** (a slowly-updated copy for the `max` term so you're not chasing a moving target). The classic remaining pitfall is the `max` operator's **overestimation bias**; Double DQN fixes it by decoupling action selection from evaluation.
