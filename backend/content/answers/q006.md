A heuristic `h(n)` is an estimate of the remaining cost from node `n` to the goal. It's the domain knowledge that turns blind search into informed search — it tells the algorithm which frontier nodes look promising so it can prioritize them instead of expanding everything uniformly.

A heuristic is **admissible** if it never overestimates the true cost-to-go: `h(n) ≤ h*(n)` for every node, where `h*` is the actual optimal remaining cost. In other words it's optimistic. Admissibility is exactly the property that guarantees A* returns an optimal solution — if the heuristic were ever too high, A* could discard the truly cheapest path because it looked expensive.

A stronger, more useful property is **consistency (monotonicity)**: `h(n) ≤ cost(n, n') + h(n')` for every neighbor `n'`. Consistency implies admissibility and additionally guarantees A* never has to re-expand a closed node, which is a real performance win. Almost every "natural" heuristic you'd design — straight-line/Euclidean distance for grids, Manhattan distance for 4-connected movement, Hamming/Manhattan for the 8-puzzle — is both.

The engineering tension: a heuristic that's closer to `h*` (more "informed") expands fewer nodes, but if you push it too hard it becomes inadmissible and you lose the optimality guarantee. The trick I reach for is pattern databases — precomputed exact costs of relaxed subproblems give tight, provably admissible values without hand-tuning.
