An SVM finds the hyperplane that separates two classes with the **maximum margin** — the widest possible gap between the boundary and the nearest points of each class. Only those nearest points, the **support vectors**, define the boundary; everything else is irrelevant, which is what makes SVMs robust and memory-light at inference. Maximizing the margin is a principled form of regularization: it favors the most confident, best-generalizing separator.

Real data isn't cleanly separable, so soft-margin SVMs add a penalty `C` that trades off margin width against misclassifications. Small `C` = wider margin, more tolerance for errors (more regularized); large `C` = fewer training errors, higher overfitting risk.

The **kernel trick** is the elegant part. Many datasets aren't linearly separable in their original space but become so in a higher-dimensional space. The trick is that the SVM's math only ever needs **dot products** between points, so you replace the dot product with a **kernel function** `K(x, x')` that computes the similarity as if you'd mapped into that high-dimensional space — *without ever materializing the coordinates*. The RBF (Gaussian) kernel effectively works in infinite dimensions. Common kernels: linear, polynomial, RBF.

In production I reach for SVMs on small-to-medium, high-dimensional problems — text classification with a linear kernel is a classic sweet spot. They don't scale well past ~100k rows (training is roughly quadratic), give no native probabilities without Platt scaling, and demand feature scaling; beyond that size I move to gradient-boosted trees.
