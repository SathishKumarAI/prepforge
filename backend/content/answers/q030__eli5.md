**Logistic regression** answers yes/no questions — "is this email spam?" — not "how much?" Picture a gauge that can only report a **probability from 0 to 1**. Plain linear regression would happily output crazy numbers like -3 or 400, useless for a yes/no. So logistic regression takes that straight-line calculation and squashes it through an **S-shaped curve** (the sigmoid) that gently bends any value into the 0-to-1 range. Read 0.8 as "80% chance it's spam," then set a cutoff — usually 0.5 — to decide the class. It also learns differently: instead of minimizing squared distance, it's trained to make its predicted probabilities match reality (log-loss). So despite the name "regression," it's really a **classifier**. Its coefficients describe how each feature nudges the odds up or down, and a softmax version handles more than two categories.
