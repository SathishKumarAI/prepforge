question: "What is the difference between Breadth-First Search (BFS) and Depth-First Search (DFS)?"

Both explore a graph, so the only real question is: *in what order do you expand the frontier?* That single choice — the data structure holding unexplored nodes — determines everything else.

**BFS uses a queue (FIFO)**: it expands the oldest-discovered node first, so it explores in rings of increasing distance from the start. Because it exhausts depth *d* before touching depth *d+1*, the first time it reaches the goal it has found a *shortest path* (in edge count). The cost: it must remember the entire current ring, so memory grows as O(b^d) — exponential in depth.

**DFS uses a stack (LIFO)**: it plunges down one branch fully before backtracking. Memory is only O(bm) — it stores a single path plus siblings — which is why it scales to deep spaces. But it sacrifices optimality and even completeness: it can dive down an infinite branch and never return.

The unifying principle: this is a **time-space and optimality trade-off** forced by the frontier ordering. BFS pays memory for guaranteed shortest paths; DFS pays optimality for tiny memory.

The non-obvious insight: they're the *same algorithm* — generic graph search — differing only by queue vs. stack. Swap the container and behavior flips entirely.
