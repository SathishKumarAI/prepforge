First diagnose *why* imbalance hurts, because it usually doesn't hurt the way people assume. A classifier minimizing average error on 99% negatives learns the trivial rule "always predict negative" — 99% accurate, useless. The real problem is a **mismatch between the loss you optimized and the cost you care about**: a missed fraud costs far more than a false alarm, but plain accuracy treats them equally.

So the principled fixes all realign objective with cost:

- **Change the metric.** Stop looking at accuracy; use precision/recall, PR-AUC, or expected cost. This changes nothing in the model but everything in how you judge it.
- **Reweight the loss.** Multiply the minority class's loss by its inverse frequency. This directly encodes "these mistakes matter more" — cleaner than resampling.
- **Resample.** Oversample the minority (SMOTE interpolates new points along minority neighbors) or undersample the majority. Note SMOTE fabricates data, so validate carefully.
- **Move the threshold.** A model outputs probabilities; the 0.5 cutoff is arbitrary. Often the best fix is simply lowering the decision threshold using a calibrated model.

Non-obvious insight: imbalance is frequently a *threshold and evaluation* problem, not a training problem. A well-calibrated model on raw data plus a cost-tuned threshold often beats elaborate resampling.
