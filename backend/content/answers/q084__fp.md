Start from the constraint that governs everything: **the viewer's attention is a fixed, tiny budget**. Working memory holds only a handful of items, and a glance lasts seconds. A dashboard is therefore an exercise in *information compression under an attention constraint*—maximize decision-relevant signal per unit of cognitive effort.

This derives the core principles. First, **every element must earn its ink**. Tufte's data-ink ratio isn't aesthetics; superfluous gridlines, 3D effects, and decoration consume attention that carries no information, lowering signal-to-noise. Remove them and the true pattern surfaces.

Second, **a dashboard answers a question, not "shows data."** Because attention is scarce, you must know the decision it serves and lead with the metric that drives it. Structure follows: headline KPIs first (the *what*), supporting breakdowns second (the *why*), detail on demand.

Third, **context converts a number into a judgment**. "Revenue: 4.2M" is inert; "4.2M, +8% vs. target, ▲ from last month" lets the brain classify it instantly as good or bad. Comparison is what makes a number *actionable*.

The non-obvious insight most people miss: a good dashboard is designed around the **pre-attentive** visual system—color, position, size are processed before conscious thought, so anomalies should *pop out* in milliseconds. If a user has to hunt for the problem, the dashboard has already failed.
