question: "What is database normalization and what problem does it solve?"

**Normalization** means organizing a database so each fact lives in exactly one place, split across tidy, linked tables.

Imagine a spreadsheet where every order also retypes the customer's full address. If that customer moves, you must hunt down and fix every single row, miss one and your data now contradicts itself (an "update anomaly"). You also can't add a new customer until they've placed an order (an "insertion anomaly"), and deleting their last order erases their address entirely (a "deletion anomaly").

Normalization fixes this by storing the address once in a customers table and just pointing to it from each order. The rules come in stages, 1NF, 2NF, 3NF, each removing another kind of duplication.

The trade-off: splitting data means questions need more joins, which can be slower. So analytics warehouses often deliberately do the reverse (denormalize) for speed. Normalization favors integrity; denormalization favors fast reads.
