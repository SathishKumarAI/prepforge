Start with the fundamental problem: we want machines to produce correct behavior in situations we cannot fully specify by hand. That single goal generates all three terms as answers at different levels of abstraction.

**AI** is the outermost answer — *any* technique that makes a machine act intelligently, including hand-written rules (a chess engine's if-then logic, A* search). No learning required; a human still supplies the intelligence.

**Machine Learning** appears the moment specification-by-hand fails. For tasks like "recognize a cat," there is no rule set a human can write. So we invert the problem: instead of programming the function, we program a *search over functions* and let data pick the one that fits. ML is AI where the rules are inferred from examples by optimizing a loss.

**Deep Learning** answers a follow-up: where do the features come from? Classical ML needs humans to hand-craft inputs. Deep nets stack layers so that *representations themselves are learned*, composing simple features into complex ones.

The non-obvious insight: these aren't three technologies but three retreats from manual specification — rules, then functions, then representations. Each subset trades human effort for data and compute.
