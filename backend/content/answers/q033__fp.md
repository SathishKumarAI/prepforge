A learning algorithm doesn't see the world — it sees the *representation* you hand it. Feature engineering is the craft of transforming raw data into inputs whose geometry makes the target easy to express. Its importance follows from a simple truth: **the same information can be trivially or impossibly learnable depending on how it's encoded.**

Consider predicting whether a point is inside a circle. In raw (x, y) coordinates the boundary is nonlinear, so a linear model fails. Add one engineered feature, x²+y², and the boundary becomes a single threshold — a linear model solves it instantly. Nothing about the information changed; only its representation did. This is the whole game: feature engineering *pre-bends the space* so a simple, low-variance model suffices.

Why it matters practically:

- It injects **domain priors** the algorithm can't discover from limited data, reducing the effective complexity needed.
- It lets you use interpretable, data-efficient models instead of throwing capacity (and overfitting risk) at the problem.
- Good features often beat fancier algorithms — signal must exist in the inputs before any model can extract it.

The unifying principle: learning = search over functions; a good representation *shrinks the space you must search* to include the answer near its "simple" region.

The non-obvious insight: deep learning didn't eliminate feature engineering — it *automated* it, learning hierarchical features from data. But that requires large data and compute; when either is scarce, hand-crafted features still win.
