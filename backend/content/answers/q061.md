A p-value is the probability of seeing data **at least as extreme** as what you observed, *assuming the null hypothesis is true*. It measures how surprising your result would be in a world where there's no real effect. Small p-value → your data is unlikely under the null → evidence to reject it. Large p-value → data is consistent with the null → you *fail to reject* (which is not the same as proving the null true). The threshold α — conventionally 0.05 — is set *before* you look.

The misconceptions are where interviews (and real analyses) go wrong, so I state them explicitly:

- It is **NOT** the probability the null is true, nor the probability your finding is a fluke. That's a different quantity requiring a prior (Bayes).
- It says **nothing about effect size**. With a huge sample, a trivial, useless difference can be wildly "significant." p<0.05 ≠ important.
- **Statistical significance ≠ practical significance.**

So in practice I never report a p-value alone. I pair it with an **effect size and a confidence interval** — the CI tells you both whether the effect is significant (does it exclude zero?) *and* how big it plausibly is, which is what a decision-maker actually needs. And I watch for **p-hacking**: running many tests or peeking until something crosses 0.05. If you test 20 things at α=0.05, you expect one false positive by chance — which is exactly why pre-registration and multiple-comparison corrections exist.
