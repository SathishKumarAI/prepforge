PCA is a linear dimensionality-reduction technique that finds a new set of orthogonal axes — the **principal components** — ordered by how much variance in the data they capture. The first component points in the direction of maximum variance, the second in the next-most-variance direction orthogonal to it, and so on. Keep the top few and you've compressed the data while retaining most of its spread.

Mechanically: center (and usually standardize) the features, compute the covariance matrix, then take its **eigenvectors** (the component directions) and **eigenvalues** (the variance each explains). In practice you do this via **SVD** on the data matrix, which is more numerically stable than forming the covariance matrix explicitly. Project the data onto the top `k` eigenvectors to get the reduced representation. The **explained-variance ratio** tells you how many components to keep — I usually pick `k` to retain 90–95% of variance, or eyeball the scree-plot elbow.

Two things people forget. **You must scale first** — PCA maximizes variance, so an unscaled large-range feature will masquerade as the dominant component regardless of importance. And PCA is **unsupervised**: it maximizes variance, not class separation, so the directions of highest variance aren't necessarily the ones that predict your label (use LDA if you want that).

I use it mostly to kill multicollinearity, speed up downstream models, and visualize high-dimensional data in 2D. The cost is interpretability — components are linear blends of original features, so you trade meaning for compactness.
