Retention asks: of the people who started together, how many keep coming back later? Think of tracking how many of each year's new club members still show up in month two, month three, and so on.

The recipe in SQL has a clear shape, whatever the exact date functions:

First, give every user a **cohort**: the month they first did anything. You find each person's earliest activity and round it to the month.

Second, list every month each user was **active**.

Third, join those together and compute the **gap**, how many months after joining each activity happened. Then count the distinct users for each (cohort, gap) pair. That builds the grid: rows are starting months, columns are months-since-joining.

Finally, turn counts into a **rate** by dividing each cell by the cohort's starting size (its month-0 count). The dialect details vary, but the pattern, assign cohort, measure the offset, count active users, then normalize, is always the same.
