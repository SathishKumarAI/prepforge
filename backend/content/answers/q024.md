Bias in AI produces systematically unfair or inaccurate outcomes for certain groups, and it almost never originates in "the algorithm" — it originates upstream. The main sources:

- **Data / historical bias** — training data reflects real-world discrimination (e.g., past hiring or lending decisions), so the model faithfully learns the inequity.
- **Sampling / representation bias** — under-represented groups get worse accuracy simply because they're scarce in the data.
- **Labeling bias** — subjective or inconsistent human annotations bake in annotator prejudice.
- **Aggregation bias** — one model optimized for overall accuracy quietly underperforms on subgroups.
- **Deployment / feedback loops** — predictions shape future data (predictive policing sends patrols where it already predicted crime, which generates more arrests there).

Addressing it is a lifecycle discipline, not a one-time fix. Start with representative data and auditing; measure with **subgroup-disaggregated metrics** and formal fairness criteria (demographic parity, equalized odds) — a single aggregate number hides the harm. Apply pre-, in-, or post-processing mitigations (reweighting, fairness constraints, per-group threshold tuning). Document with datasheets and model cards, and keep humans in the loop for high-stakes decisions.

Two hard truths I always flag: fairness definitions mathematically conflict — you generally can't satisfy demographic parity and equalized odds simultaneously — so you must make an explicit, stakeholder-informed choice about which harm matters. And bias is never "solved," only monitored; models drift, so I set up continuous fairness dashboards the same way I'd monitor latency or accuracy.
