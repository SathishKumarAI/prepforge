question: "Why are Convolutional Neural Networks (CNNs) well-suited for images?"
topic: "AI"
model: "Claude Code (authored)"
sources: []
---

Start from the problem: an image is a grid of pixels, and a naive fully-connected network treats each pixel as an independent input. For a modest image that means millions of weights in the first layer alone — statistically hopeless, because you'd need astronomically more data to constrain them.

The insight is that images have **structure we can bake in as priors** rather than learn from scratch. Two facts dominate: *locality* (a pixel's meaning depends mostly on its neighbors, not on pixels across the frame) and *translation invariance* (an edge is an edge whether it appears top-left or bottom-right). Convolution encodes both. A small filter slides across the image, sharing one set of weights everywhere — this is **weight sharing**, which slashes parameters and enforces the prior that the same feature detector applies at every position.

Stacking convolutions builds a **hierarchy**: edges → textures → parts → objects, mirroring how visual information composes. Pooling adds robustness to small shifts.

The non-obvious point: CNNs work not because they're more powerful than dense nets, but because they're *deliberately weaker* — the architectural constraints match the geometry of natural images, so the hypothesis space is small and learnable.
